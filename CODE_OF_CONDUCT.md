## 试卷

样式比较复杂, 具体参照 [信息安全数学基础（一）/2023-2024学年下学期期末_含答案.md](https://raw.githubusercontent.com/BetterECNU/SharedCourses/refs/heads/main/docs/courses/软件工程学院/信息安全数学基础（一）/2023-2024学年下学期期末_含答案.md)

### 文件命名

- 试卷文件名按学年、学期、考试类型命名, 例如 `2023-2024学年下学期期末.md`。
- 如试卷含答案, 文件名后加 `_含答案`, 例如 `2023-2024学年下学期期末_含答案.md`。
- 同一学年、同一学期、同一考试类型下存在 A/B 卷、不同教师版本或其他并行版本时, 不要在文件名中区分卷别或教师, 应合并到同一个 Markdown 文件中。例如:

``` markdown
---
title: 2023-2024学年下学期期末_含答案
---

## 2023-2024学年下学期期末试卷（A）

...

***

## 2023-2024学年下学期期末试卷（B）

...

***

## 2023-2024学年下学期期末试卷（A）——张佳秋

...
```

- 课程页面中也只挂一个链接:

``` markdown
- [2023-2024学年下学期期末试卷](./2023-2024学年下学期期末_含答案)
```

### 内容格式

- 试卷正文应尽量尊重原卷题面, 不要擅自添加原卷没有的小标题。
- 原卷中有大题标题时, 使用 `### 一、选择题`、`### 二、填空题` 等格式。
- 原卷中只有题号时, 直接使用 `1.`、`2.` 等编号即可。
- 卷别、教师或版本差异写在 Markdown 正文标题中, 例如 `## 2023-2024学年下学期期末试卷（A）`、`## 2023-2024学年下学期期末试卷（A）——张xx`。
- 同一大题内的题目之间使用缩进 4 个空格的分隔线 `    ***`；大题之间使用顶格分隔线 `***`。例如:

``` markdown
1. 第一题题面

    ***

2. 第二题题面

***

### 二、填空题
```

- 答案放在对应题目下方的折叠块中:

``` markdown
<details>
<summary>解：</summary>

答案内容

</details>
```

- 选择题选项建议使用 A/B/C/D 分行书写; 数学公式使用 LaTeX。
- 不确定或来自回忆的题目, 应在正文中用 `### 说明` 标明来源不确定性。

### 图片处理

- 题目中不可缺少的图片, 如拓扑图、时序图、协议报文格式、坐标图、表格截图等, 应放入课程目录下的 `images/` 目录。
- 图片文件名应能对应试卷、卷别和题号, 例如:

``` text
images/2023-2024学年下学期期末_A_第4题.png
images/2023-2024学年下学期期末_B_第6题.png
```

- Markdown 中使用相对路径引用图片:

``` markdown
![A卷第4题图](./images/2023-2024学年下学期期末_A_第4题.png)
```

- 页眉、学校 logo、空白答题框、装饰线等非题目必要内容通常不需要保留。

有搞不懂样式怎么写的可以询问开发团队
